# API Dapic

Bem‑vindo à API do **Dapic**, um ERP SaaS focado em fábricas de confecção têxtil e lojas de roupas. Esta API RESTful oferece endpoints para gerenciar clientes, produtos, pedidos, estoque e outras operações essenciais do seu negócio.

- **Base URL:** [https://api.dapic.com.br](https://api.dapic.com.br/).

- **Autenticação:** Bearer Token no header `Authorization: Bearer`

- **Formato de dados:** JSON (request & response)

- **Rate limit:** 100 requisições/minuto por endpoint e chave de API

- **Versão atual:** 1.0 (primeira versão — sujeito a alterações)

Esta documentação tem por finalidade auxiliar a integração de parceiros com a nossa API.

API foi desenvolvida para fornecer informações precisas e atualizadas sobre esses aspectos, mas **não permite** a **inclusão** ou **modificação** de dados.

# Autenticação

A autenticação é feita usando um token de acesso padrão JWT( [**OAuth 2.0**](https://oauth.net/2/)). Este token tem **duração de 1 dia**. Recomendamos armazená-lo na aplicação enquanto estiver válido, para evitar rejeições por uso indevido.

### Obtenção do Token no Dapic

Para gerar o **Bearer Token** **`(access_token)`**, é necessário informar o **Token de integração**, que deve ser gerado no cadastro da [Empresa](https://dapic.webpic.com.br/admin/empresa) dentro do sistema [**DAPIC**](https://dapic.webpic.com.br/) **,** juntamente com o **Identificador da empresa** utilizado para autenticar no Dapic.

![](https://content.pstmn.io/fe00593c-5979-49c9-b6c6-0993ca6fece6/aW1hZ2UucG5n)

# **Paginação**

A API possui suporte a paginação que pode ser manipulada pelos parâmetros **`Pagina`** e **`RegistrosPorPagina`**, com um máximo de 200 registros por página.

# **Ordenação**

A API possui suporte para ordenação de registros que pode ser realizado pelos parâmetros **`TipoOrdenacao`** e **`OrdenarPor`**.

# Limite de requisições

Serão permitidas até **100 requisições** dentro de um intervalo de **1 minuto**, para um mesmo endpoint. Ao atingir o limite, a API retornará o HTTP\_CODE 429, informando " _Too Many Requests_".

PARAMS

DataInicial

2024-01-01

Representa a data inicial dentro do periodo informado para a filtragem.

DataFinal

2024-06-01

Representa a data final dentro do período informado para a filtragem.

FiltrarPor

Representa a qual data o período informado irá filtrar, no caso da fatura, possuimos três tipos:

- 0 = Data de emissão
- 1 = Data de entrega
- 2 = Data de fechamento
- 3 = Data modificação

OrdenarPor

Representa a ordenação a qual os registros serão apresentados. Aqui fornecemos as opções de:

- 0 = Data de emissão
- 1 = Identificador do registro (ID)
- 2 = Cliente
- 3 = Código da fatura
- 4 = Código externo da fatura
- 5 = Valor líquido
- 6 = Status da fatura
- 7 = Data de modificação

TipoOrdenacao

Representa o modo de ordenação, que pode ser:

- 0 = Decrescente
- 1 = Crescente

Status

Representa o "Status" e para a fatura temos os seguintes status:

- 0 = Aberto
- 4 = Cancelado
- 5 = Fechado

Pagina

Página atual da consulta valor padrão 1

RegistrosPorPagina

Registros por página valor padrão 10

4

- 0 = Data de fechamento
- 1 = Data de cadastro

10

Status

Cidade

Estado

Vendedor

Cor

Tamanho

Desativados

Controla a exibição de produtos com grade ou cor desativada no sistema, deve ser informado "true" ou "false".

SaldoZerado

Controla a exibição de produtos cujo o saldo em estoque está igual a zero, deve ser informado "true" ou "false".

Página atual da consulta valor padrão 1
